Доступ


Основной документ для внесения в систему и редактирования хранимой информации пользователя. В документе производится связь классификатора, группы и вида данных. На основании выбранного вида данных в карточке доступа пользователю предлагается заполнить свойства, относящиеся к виду. При этом тип значения свойства будет использоваться тот, который был указан в свойстве вида данных.

Документ является регистратором хранимой информации в разрезе каждого доступа.

Пользователь имеет возможность в любое время эксплуатации программы изменить свойства вида данных в каждом документе.

Документ заполняется пользователем на его усмотрение и в произвольном порядке, однако обязательно нужно указать классификатор, группу и вид данных. Все свойства доступа являются необязательными для заполнения.

Статус доступа программа устанавливает автоматически, выбирая из вариантов «Актуальный» и «Неактуальный». Если доступ помечен на удаление, то статус устанавливается «Неактуальный», иначе статус используется «Актуальный».

Удалять документы доступа из списка запрещено. Не рекомендуется, однако допустимо установить пометку на удаление для каждого отдельного доступа и удалить его с помощью стандартной обработки «Удаление помеченных объектов (Стандартные)» после прохождения контроля ссылочной целостности.
