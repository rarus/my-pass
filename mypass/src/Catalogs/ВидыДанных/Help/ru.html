Виды данных

Основной справочник для хранения используемой классификации хранимой информации по виду. Например, следующие виды данных могут быть в программе:

§  «Сайт»;

§  «База данных»;

§  «Хранилище»;

§  «VPN»;

§  «Телефон»;

§  «Адрес»;

§  «Электронная почта»;

§  а также любые иные варианты, не являющиеся дублированием классификатора и групп данных.

Справочник видов данных является пользовательским классификатором третьего уровня. Разбивка информации выполняется на виды данных, созданные пользователем самостоятельно.

При первом запуске приложения пользователю предлагается выполнить начальное заполнение видов данных, а именно, добавить стандартные элементы справочника с предварительно заполненными свойствами данных.

В каждом элементе справочника пользователь может добавить произвольное количество свойств для удобного хранения информации касательно доступов, паролей и иных данных. Например, в виде «Электронная почта» можно добавить свойства «Сервис», «Логин», «Пароль»; в виде «База данных» можно добавить свойства «Сервер», «Имя базы», «Логин», «Пароль», «Версия платформы»; в виде «Телефон» можно добавить свойства «ФИО / Наименование компании», «Телефон», «ФИО контактного лица» и так далее.

См. Свойства видов данных

Пользователь имеет возможность в любое время эксплуатации программы изменить список свойств вида данных. При этом новые данные будут создаваться согласно новому списку свойств.

Если разделение информации не предполагается на виды данных (например, программа эксплуатируется как телефонный справочник), достаточно создать один универсальный элемент, например, «Общий вид данных», и использовать его во всех доступах. В таком случае вся информация будет храниться без разделения по видам данных.

Справочник заполняется пользователем на его усмотрение и в произвольном порядке. Ограничений количества создания видов данных в программе не установлено. Наименование у элемента справочника обязательно для заполнения и должно быть уникальным.

В элементе справочника можно указать уникальный цвет, который будет использоваться для выделения группировки с этими данными в рабочем месте «Мои доступы и пароли» в «современном» интерфейсе.

Удалять элементы справочника из списка запрещено. Не рекомендуется, однако допустимо установить пометку на удаление для отдельного элемента и удалить его с помощью стандартной обработки «Удаление помеченных объектов (Стандартные)» после прохождения контроля ссылочной целостности.
